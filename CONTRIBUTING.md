# Contributing

Read the README, About, and Roadmap before contributing.

Use an Issue before making a large architecture, format, compatibility, or integration change. Keep each pull request focused and include the problem, intended result, examples, validation steps, compatibility impact, and documentation updates.

Changes that generate repository files should include deterministic fixtures showing the input project and expected generated output.

Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review of submitted material. Do not include private repositories or material that you do not have permission to publish.
